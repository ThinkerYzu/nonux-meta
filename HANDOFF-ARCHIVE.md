2. **[Session 78](logs/session-78-7.8c-yield-loop-migration.md)** (2026-04-29) — slice 7.8c: yield-loop migration + busybox-patch revert.  Closes slice 7.8 and Phase 7.  All three v1 yield-loop call sites — `sys_read` CHANNEL arm (slice 7.6d.N.6b), `sys_wait` (slice 7.4b), `nx_console_read` (slice 7.6d.N.final.a) — now block via `nx_waitq_wait_unless` on the appropriate object's waitq. 59. **[Session 3: Component-spawned threads and thread-local slot dereferences](logs/session-03-component-spawned-threads.md)** (2026-04-18) — Design-only session, no code changes:
   - **Spec:** SPEC R18 gained a new sub-bullet "Component-spawned threads and thread-local dereferences." Components may spawn their own worker threads for CPU-bound work that fits neither the bounded-handler rule nor `dedicated` mode. Two rules: (1) dereferenced slot pointers (the `impl*` from `slot_resolve()`) are thread-local — a dispatcher handler must not pass a resolved pointer to a worker; workers carry slot refs instead and enqueue messages that a dispatcher will resolve; (2) the component must declare a manifest `pause_hook` and drain its own workers during the pause protocol's cutoff step. Preference order: `dedicated` first, component-spawned threads only when the workload doesn't fit a simple message-queue shape.

**7.6d.N.final.a — RX IRQ + console RX ring + blocking
`nx_console_read` + termios/ioctl stubs.**

Wire PL011 RX into the kernel.  PL011 lives at MMIO
`0x09000000` on QEMU virt; RX is SPI 1 (= IRQ 33).  Steps:

1. Extend `framework/console.{h,c}` with a fixed-size byte
   ring (256 bytes is plenty — single line of typed input).
   Single-producer (PL011 RX IRQ) / single-consumer
   (`nx_console_read` from EL0 syscall).  Use plain head/tail
   indices with `_Atomic` updates; the IRQ runs with IRQs
   masked at the GIC, so no further synchronisation needed.
2. New `nx_console_rx_isr(void *data)` IRQ handler — drain
   PL011 RX FIFO into the ring, drop bytes if ring is full.
   Register via `irq_register(33, nx_console_rx_isr, NULL)` +
   `gic_enable(33)` from `framework/console_init` (called from
   `bootstrap.c` after GIC is up).
3. Enable PL011 RX FIFO + RX IRQ in the UART driver.  Set
   `UARTIMSC.RXIM` and `UARTIFLS.RXIFLSEL` to "1/8 full"
   (interrupt on first byte).  Add to `components/uart_pl011/`
   init or to a `uart_pl011_rx_enable()` helper.
4. `nx_console_read(void *buf, size_t cap)` becomes a
   yield-loop: while ring is empty and signal not pending,
   `nx_task_yield()`; on wake, drain up to `cap` bytes from
   the ring + return.  No line discipline yet — ash drives
   line-editing in raw mode via the cmdedit path (it's
   already linked into busybox).
5. New `NX_SYS_IOCTL = 39` in `framework/syscall.c`.  Recognise:
   - `TCGETS` (`0x5401`) → fill 19-byte `struct termios` with
     zeros, return 0.
   - `TCSETS` / `TCSETSW` / `TCSETSF` (`0x5402` / `0x5403` /
     `0x5404`) → ignore, return 0.
   - `TIOCGWINSZ` (`0x5413`) → fill 4×u16 `{ws_row=24,
     ws_col=80, ws_xpixel=0, ws_ypixel=0}`, return 0.
   - other cmds → `-ENOTTY` (Linux value `-25`).
6. musl translation: `__NR_ioctl (29) → NX_SYS_IOCTL` in both
   `arch/aarch64/syscall_arch.h` and
   `src/thread/aarch64/syscall_cp.s`.
7. ktest `posix_console_rx`: spawn an EL0 program that calls
   `read(0, buf, 4)`; before drop_to_el0, schedule the kernel
   side to inject 4 bytes via a new
   `nx_console_test_inject_bytes(const char *, size_t)` helper

   came back.  Plus a `posix_isatty` ktest that calls
   `isatty(0)` (which musl implements via `tcgetattr`) and
   asserts the result is 1.

**7.6d.N.final.c — Ctrl-C → SIGINT + Ctrl-D EOF + signal
trampoline.**

(Note: this trampoline approach was deferred as `c-full`;
Session 71 shipped a polled `c-minimal` instead.  Spec
preserved here for the eventual focused future session.)

Promote slice N.12's no-op `sigaction` to actually-installs-
handler.  Per-process `sigaction[NSIG]` table keeps the
handler PC + flags + sa_mask.  Console RX ISR watches for
`0x03` (Ctrl-C) and `0x04` (Ctrl-D):

- `0x03` → `nx_signal_send(current_process, SIGINT)`; byte
  is NOT pushed to the ring.
- `0x04` at start of read (ring empty when the read started)
  → return 0 = EOF; otherwise pushed as a normal byte.

The kernel→user signal trampoline sits at the EL0-return
edge in `framework/syscall.c`.  When returning to EL0 with a
pending signal that has a handler installed:

1. Save the trap frame to a slot at the top of the EL0 user
   stack.
2. Set `tf->elr = sa_handler_pc; tf->x[0] = signo;
   tf->sp = sp_after_frame; tf->lr = sigreturn_trampoline_pc`.
3. `sigreturn_trampoline_pc` is a tiny EL0-side stub
   (single instruction `svc #0` with x8 = `NX_SYS_RT_SIGRETURN`)
   embedded in the user-window backing at a known offset by
   `sys_exec` / `nx_process_create`.
4. New `NX_SYS_RT_SIGRETURN = 40` restores the saved trap
   frame and re-enters the interrupted code.

ktest `posix_console_sigint`: EL0 program installs a SIGINT
handler that increments a TLS counter, then calls
`read(0, buf, 1)`; kernel injects `0x03`; EL0 program
asserts handler ran + `read` returned `-1` with `errno =
EINTR`.

**Exit:** Three new ktests pass (SIGINT delivery, EOF on
Ctrl-D, handler runs concurrently with read).  `make test`
