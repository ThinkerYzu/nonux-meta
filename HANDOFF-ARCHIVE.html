7.6d.N.final.c — Ctrl-C → SIGINT + Ctrl-D EOF + signal trampoline.

(Note: this trampoline approach was deferred as c-full; Session 71 shipped a polled c-minimal instead. Spec preserved here for the eventual focused future session.)

Promote slice N.12’s no-op sigaction to actually-installs- handler. Per-process sigaction[NSIG] table keeps the handler PC + flags + sa_mask. Console RX ISR watches for 0x03 (Ctrl-C) and 0x04 (Ctrl-D):

  • 0x03 → nx_signal_send(current_process, SIGINT); byte is NOT pushed to the ring.

  • 0x04 at start of read (ring empty when the read started) → return 0 = EOF; otherwise pushed as a normal byte.

The kernel→user signal trampoline sits at the EL0-return edge in framework/syscall.c. When returning to EL0 with a pending signal that has a handler installed:

  1. Save the trap frame to a slot at the top of the EL0 user stack.

  2. Set tf->elr = sa_handler_pc; tf->x[0] = signo; tf->sp = sp_after_frame; tf->lr = sigreturn_trampoline_pc.

  3. sigreturn_trampoline_pc is a tiny EL0-side stub (single instruction svc #0 with x8 = NX_SYS_RT_SIGRETURN) embedded in the user-window backing at a known offset by sys_exec / nx_process_create.

  4. New NX_SYS_RT_SIGRETURN = 40 restores the saved trap frame and re-enters the interrupted code.

where script.txt is a list of newline-terminated commands ending with exit. Captured stdout is printed.

New make test-interactive: builds with --busybox-init, runs three canned scripts (echo hi, ls /, exit), diffs each against tools/expected/<name>.out. Wired into the top-level make test once stable (separate target initially to avoid CI flakes from QEMU stdin handling).
