### Deferred — actionable when a workload demands

2. **7.6d.N.final.c-full — full signal-handler dispatch trampoline.**  Promote `sys_rt_sigaction` from no-op stub to actually-installs-handler (per-process `sigaction[NSIG]` table); EL0-return path pushes a synthetic frame on the user stack, sets PC to the handler, sets x0 = signo, sets LR to an EL0 trampoline (or musl's `sa_restorer`) that issues `NX_SYS_RT_SIGRETURN` to restore.  Switches Ctrl-C from "kill everyone" to "post SIGINT, deliver via handler".  Estimated ~200-300 lines kernel + ktest using mock-RX 0x03 injection.  v1 minimal Ctrl-C / Ctrl-D side channels (slice 7.6d.N.final.c-minimal, Session 71) cover the scripted smoke tests; .c-full lands when interactive UX needs handler dispatch.

3. **7.6d.N.16 — Drop in-handle generation encoding + POSIX-fd-to-slot alignment.**  Originally a 30-line paydown for slice 7.6d.N.11's `sys_fcntl` band-aid (replace `(generation << 8) | (idx + 1)` with `idx + 1`); slice 7.6d.N.15's investigation showed the proper fix needs a non-zero `NX_HANDLE_INVALID` sentinel (e.g. `0xFFFFFFFF`) so encoded value 0 can name fd 0 — ripples through every `NX_HANDLE_INVALID` comparison and the slice-5.3 stale-handle invariants (which would then assert via the in-table generation field rather than encoded-value mismatch).  Lands when a workload demands `exec 3< file` or similar literal-fd-3+ idioms.  ~30 lines + slice-5.3 test churn.

4. **Reap-on-wait.**  Have `sys_wait` call `nx_process_destroy(child)` after collecting exit status, freeing the process struct + handle table + MMU address space + 8 MiB user backing.  Today every test leaks the full child state: `sched_rr_purge_user_tasks` only unlinks tasks from the runqueue, never frees them.  Slice 7.6c.4 bumped `NX_PROCESS_TABLE_CAPACITY` + `MMU_MAX_ADDRESS_SPACES` 16 → 32 to paper over the leak; slice 7.6d.N.2 bumped 32 → 64; slice 7.6d.N.15 bumped 64 → 128.  Lands when the next round of escalations refills the 128-slot waterline (cap is plenty for now).

### Non-blocking opportunistic follow-ups

5. **Vector I/O syscalls beyond `writev` / `readv`.**  Slice 7.6c.3c + 7.6d.N.6a wired what musl's `__stdio_write` / `__stdio_read` need; the natural follow-on family returns `-ENOSYS`:

4. **[Session 78](logs/session-78-7.8c-yield-loop-migration.md)** (2026-04-29) — slice 7.8c: yield-loop migration + busybox-patch revert.  Closes slice 7.8 and Phase 7.  All three v1 yield-loop call sites — `sys_read` CHANNEL arm (slice 7.6d.N.6b), `sys_wait` (slice 7.4b), `nx_console_read` (slice 7.6d.N.final.a) — now block via `nx_waitq_wait_unless` on the appropriate object's waitq.  New `nx_waitq_wait_unless(wq, budget_ns, pred, ctx)` re-evaluates `pred(ctx)` *inside* the IRQ-disabled + preempt-disabled critical section that registers the caller; if it returns non-zero, the caller is NOT enqueued and the function returns `NX_OK` immediately (treats the predicate as an already-fired wake).  This is the lost-wakeup-safe wait pattern — a wake that fires between the caller's outer `cond` check and the inner predicate check is caught by the predicate; on single-CPU the gap is real because timer interrupts can preempt.  Internal restructuring: shared body of `wait_with_deadline` + `wait_unless` extracted into a static `waitq_register_and_wait_locked` helper.  `struct nx_process` grows `struct nx_waitq exit_waitq` (initialised in `nx_process_create`; statically self-pointing for `g_kernel_process`).  `nx_process_exit` walks up to `parent_pid` and calls `nx_waitq_wake_all(&parent->exit_waitq)` after the handle-table cleanup but before the terminal `wfe` loop.  Processes spawned outside fork (`parent_pid == 0`) skip the wake — nothing waits on them.  `sys_wait` migrates to `nx_waitq_wait_unless(&caller->exit_waitq, 0, pred, ctx)` for both `pid != -1` (predicate: `target->state == EXITED`) and `pid == -1` (predicate: reapable EXITED child OR no children at all) branches.  Two new static-inline predicates above `sys_wait`, both guarded `#if !__STDC_HOSTED__`.  `sys_read` CHANNEL arm: kstack `struct nx_waitq` + `struct nx_pollset_listener` pair, listener registered on the endpoint's pollset_listeners list; loop body unchanged except `nx_task_yield()` → `nx_waitq_wait_unless(&read_wq, 0, sys_read_channel_ready_pred, obj)`.  Predicate returns non-zero when `nx_channel_endpoint_depth(e) > 0` OR `nx_channel_endpoint_peer_closed(e)` — exactly the "would not return EAGAIN" condition.  No producer-side wake additions: the slice 7.8b producer wakes (`nx_channel_send` + `_close` walking `pollset_listeners`) already fan out to every registered listener.  `nx_console_read`: same pattern — kstack waitq + listener registered on `g_console_pollset_listeners`; predicate (`console_read_ready_pred`) covers ring-non-empty OR Ctrl-D EOF armed. Each call site gets its own private kstack waitq rather than a per-endpoint shared one, which avoids the thundering-herd issue (a per-endpoint `read_waitq` shared across N concurrent readers would wake all N for one byte; the per-listener fan-out wakes only the active waiters).  `third_party/busybox/libbb/lineedit.c` reverts to upstream form — the slice 7.6d.N.final.e `fflush(stdout)` patch + its 20-line comment block deleted (~25 lines total).  `put_prompt_custom` reverts to plain `fputs_stdout(prompt) + cursor = 0 + cmdedit_y/x assignments`.
