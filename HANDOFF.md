2. **[Session 78](logs/session-78-7.8c-yield-loop-migration.md)** (2026-04-29) ŌĆö slice 7.8c: yield-loop migration + busybox-patch revert.  Closes slice 7.8 and Phase 7.  All three v1 yield-loop call sites ŌĆö `sys_read` CHANNEL arm (slice 7.6d.N.6b), `sys_wait` (slice 7.4b), `nx_console_read` (slice 7.6d.N.final.a) ŌĆö now block via `nx_waitq_wait_unless` on the appropriate object's waitq.  New `nx_waitq_wait_unless(wq, budget_ns, pred, ctx)` re-evaluates `pred(ctx)` *inside* the IRQ-disabled + preempt-disabled critical section that registers the caller; if it returns non-zero, the caller is NOT enqueued and the function returns `NX_OK` immediately (treats the predicate as an already-fired wake).  This is the lost-wakeup-safe wait pattern ŌĆö a wake that fires between the caller's outer `cond` check and the inner predicate check is caught by the predicate; on single-CPU the gap is real because timer interrupts can preempt.  Internal restructuring: shared body of `wait_with_deadline` + `wait_unless` extracted into a static `waitq_register_and_wait_locked` helper.  `struct nx_process` grows `struct nx_waitq exit_waitq` (initialised in `nx_process_create`; statically self-pointing for `g_kernel_process`).  `nx_process_exit` walks up to `parent_pid` and calls `nx_waitq_wake_all(&parent->exit_waitq)` after the handle-table cleanup but before the terminal `wfe` loop.  Processes spawned outside fork (`parent_pid == 0`) skip the wake ŌĆö nothing waits on them.  `sys_wait` migrates to `nx_waitq_wait_unless(&caller->exit_waitq, 0, pred, ctx)` for both `pid != -1` (predicate: `target->state == EXITED`) and `pid == -1` (predicate: reapable EXITED child OR no children at all) branches.  Two new static-inline predicates above `sys_wait`, both guarded `#if !__STDC_HOSTED__`.  `sys_read` CHANNEL arm: kstack `struct nx_waitq` + `struct nx_pollset_listener` pair, listener registered on the endpoint's pollset_listeners list; loop body unchanged except `nx_task_yield()` ŌåÆ `nx_waitq_wait_unless(&read_wq, 0, sys_read_channel_ready_pred, obj)`.  Predicate returns non-zero when `nx_channel_endpoint_depth(e) > 0` OR `nx_channel_endpoint_peer_closed(e)` ŌĆö exactly the "would not return EAGAIN" condition.  No producer-side wake additions: the slice 7.8b producer wakes (`nx_channel_send` + `_close` walking `pollset_listeners`) already fan out to every registered listener.  `nx_console_read`: same pattern ŌĆö kstack waitq + listener registered on `g_console_pollset_listeners`; predicate (`console_read_ready_pred`) covers ring-non-empty OR Ctrl-D EOF armed. Each call site gets its own private kstack waitq rather than a per-endpoint shared one, which avoids the thundering-herd issue (a per-endpoint `read_waitq` shared across N concurrent readers would wake all N for one byte; the per-listener fan-out wakes only the active waiters).  `third_party/busybox/libbb/lineedit.c` reverts to upstream form ŌĆö the slice 7.6d.N.final.e `fflush(stdout)` patch + its 20-line comment block deleted (~25 lines total).  `put_prompt_custom` reverts to plain `fputs_stdout(prompt) + cursor = 0 + cmdedit_y/x assignments`. New `framework/pollset.h` defines a shared `struct nx_pollset_listener { struct nx_list_node node; struct nx_waitq *waitq; }` + an inline `nx_pollset_wake_all(list)` that walks a per-object listener list and calls `nx_waitq_wake_all` on each entry's parent waitq.  `struct nx_channel_endpoint` grows `pollset_listeners` (initialised in `nx_channel_create`); `framework/console.c` gains a singleton `g_console_pollset_listeners` (statically self-pointing). Channel readiness: POLLIN if recv would not return NX_EAGAIN (ring non-empty OR peer closed); POLLOUT if peer's ring has space and neither side closed; POLLHUP whenever peer closed (always set, independent of `want`).  Console readiness: POLLIN if `rx_count() > 0` OR `g_eof_pending` armed; POLLOUT always set.  Producer wakes: `nx_channel_send` (after pushing to peer's ring ŌåÆ wake peer's listeners), `nx_channel_endpoint_close` (after last close ŌåÆ wake both endpoints' listeners), `nx_console_rx_isr` (after draining FIFO into ring + arming control flags ŌåÆ wake if a real byte was pushed OR EOF armed; Ctrl-C alone does not wake), `nx_console_test_inject_bytes` + `nx_console_test_inject_eof` (host-test parity).  `sys_ppoll` (~140 lines in `framework/syscall.c`): validates `nfds <= NX_PPOLL_MAX_FDS = 32`; decodes optional `struct timespec` (16 bytes on aarch64; budget = `tv_sec * 1e9 + tv_nsec`); copy_from_user the pollfd array; walks handles with STDIN special-case (encoded fd 0 ŌåÆ slot 2, mirroring `sys_read`) and per-type registration (`CONSOLE` / `CHANNEL` register listeners; `FILE` / `DIR` skip ŌĆö always-ready; bad lookup ŌåÆ `NX_POLLNVAL`); **registers listeners BEFORE the initial readiness check** (lost-wakeup-safe ordering ŌĆö a wake during the registrationŌåÆcheck window reaches the empty pollset waitq as a no-op, but the underlying state is mutated, so the readiness check catches it); if any fd ready or `timeout` is zero (non-blocking), skip the sleep; otherwise blocks via `nx_waitq_wait_with_deadline(&pollset_wq, budget_ns)`; wake (NX_OK) or deadline expiry (NX_EDEADLINE) both fall through to a re-check; unregister every listener; copy_to_user revents; return ready count.  Used `nx_process_current()` rather than `nx_task_current()` for the handle-table lookup so the host build (which only includes `core/sched/task.h` under `#if !__STDC_HOSTED__` in syscall.c) compiles cleanly.  Dispatch table gains `[NX_SYS_PPOLL] = sys_ppoll`; `NX_SYSCALL_COUNT` 41 ŌåÆ 42.  musl translation: `case 73: return 41;` in `arch/aarch64/syscall_arch.h`'s `__nx_translate()`, plus `cmp x1, #73; b.eq .Lnx_ppoll` + `.Lnx_ppoll: mov x8, #41; b .Lnx_run` in `src/thread/aarch64/syscall_cp.s`.
